# 活動當天操作手冊 🐟

## 事前準備（前一天）

1. 筆電裝好 Node.js（v18+），專案資料夾複製到筆電
2. 在專案資料夾開終端機：`npm install`（只需做一次，需網路）
3. 試跑：`npm start` → 瀏覽器開 http://localhost:3000 看到水族箱即成功
4. 準備背景圖（建議 3~6 張、橫式 1920×1080，海底照片或圖畫）
   - 開 http://localhost:3000/admin/ → 「上傳背景圖」
5. 清掉測試魚：控制台魚清單逐隻刪除，或關掉伺服器後執行：
   `rm data/fish/*.png data/state.json`（背景圖會保留）
6. 列印或抄下「拍照小技巧」貼在上傳桌（上傳頁裡也有顯示）

## 當天開機順序

1. 筆電接投影機（建議設「延伸桌面」或直接鏡像）
2. **網路擇一：**
   - A. 筆電和手機連同一個校內 Wi-Fi（手機需能連到筆電 IP）
   - B. 校網不通就用筆電「網際網路共享」開熱點，手機連筆電熱點
   - C. 或用手機開熱點，筆電 + 其他手機都連它
3. **雙擊專案資料夾裡的「啟動水族箱.command」**（免打指令；會自動清掉舊程序、開伺服器、自動打開入口頁。跳出的黑色視窗不要關）。要停止就雙擊「停止水族箱.command」或關掉那個視窗
4. 入口頁點「進入水族箱」到 http://localhost:3000/display/ → 按 **F11 全螢幕**（或滑鼠雙擊畫面）
5. 工作人員手機/另一台電腦開 http://localhost:3000/admin/
6. 控制台按「⛶ 全螢幕顯示 QR Code」→ 先投影給大家掃，或印出來貼在桌上
7. 要音效的話：在投影頁**點一下滑鼠**（瀏覽器規定要點過才能出聲），控制台勾「環境音效」

| 一次處理多隻 | 勾選魚 →「批次隱藏/顯示/刪除」 |
| 小朋友找自己的魚 | 上傳過的手機會有「🔍 找『名字』」鈕，按了魚游到中央亮相 |

## 常見問題

- **手機掃 QR 連不上** → 手機和筆電必須在同一個網路；確認後重開 admin 頁（QR 會用最新 IP）。校園 Wi-Fi 常隔離裝置（AP isolation），連不上就改用熱點方案 B/C。
- **去背效果差** → 重拍：紙攤平、光線均勻不要有影子、垂直拍。框選時框緊一點。
- **投影頁當掉/不動** → 瀏覽器重新整理（F5）即可，魚和設定都在伺服器，不會掉。
- **筆電休眠害投影黑掉** → 系統設定把「螢幕休眠/睡眠」關掉，電源接著。
- **5 台以上手機同時傳** → 沒問題，實測 25 張併發 1 秒內處理完。
- **想全部重來** → 關伺服器 → `rm data/fish/*.png data/state.json` → 重啟。

## 打包給團隊（Mac / Windows）

要交給團隊時，雙擊 `scripts/打包給團隊.command`，把桌面產出的 `FishAquarium_YYYYMMDD.zip` 給對方。

打包規則：
- 交付 ZIP 一律使用 ASCII/英文檔名：`FishAquarium_YYYYMMDD`、`app_files`、`start-windows.bat`、`start-mac.command`、`INSTALL.txt`。中文只保留在畫面提示與教學內容，避免 Windows 批次檔在中文雲端路徑下抓不到自身所在資料夾。
- ZIP 不包含 `node_modules`。Mac 與 Windows 的元件不同，尤其 `sharp` 這類原生模組不能跨平台直接複製。
- 對方第一次雙擊啟動檔時，會自動用自己的電腦重新安裝元件，所以第一次需要網路。
- 如果對方拿到舊包或資料夾裡殘留壞掉的 `node_modules`，新版 Mac / Windows 啟動檔會先檢查元件可不可用；不可用時會自動修復，必要時清掉重裝。
- 若 Windows 顯示 `Invalid package config ... node_modules\express\package.json`，代表那份依賴資料夾已損壞或不是這台電腦可用的版本。請用新版 `打包給團隊.command` 重新產生 ZIP，再讓對方解壓後雙擊啟動。
- Windows 啟動檔會先把程式複製到 `%LOCALAPPDATA%\MiaSchoolFishtank\app` 再執行，避免在 Google Drive、USB、網路磁碟或中文雲端路徑裡安裝 `node_modules` 造成套件損壞。活動資料會存在這個本機資料夾。
- 如果 Windows 顯示 `Cannot find app_files\package.json`，通常是使用者只複製了 `start-windows.bat`，或解壓後移動了檔案。請重新解壓整包 ZIP，保持 `start-windows.bat` 與 `app_files` 在同一層。

## Railway 部署：更版不洗掉魚/背景（重要）

Railway 預設檔案系統是「暫時的」，每次 deploy 會清空 → 魚和背景不見。
解法：掛一顆 **Volume（持久磁碟）**，讓資料存在它裡面。

1. Railway 專案 → 該服務 → **Variables** → 新增
   `DATA_DIR = /data`
2. 同服務 → **Settings → Volumes** → **New Volume**，Mount path 填 `/data`
3. Redeploy。之後上傳的魚/背景都存在 Volume，更版不會洗掉。

（程式已支援：`server/state.js` 讀 `process.env.DATA_DIR`，沒設就用本機 `data/`。）
⚠️ 已存在 Railway 舊環境的魚（在暫時磁碟上）救不回，掛 Volume 後重新上傳。

## 分享給遠端朋友測試（活動前）

### 固定網址（推薦，免費 ngrok 帳號）

ngrok 免費帳號提供 1 個固定 Dev Domain。網址會固定，但名稱由 ngrok 自動配發，不能自選。

第一次設定：
1. 註冊 / 登入免費 ngrok 帳號：https://dashboard.ngrok.com/
2. 到 Authtoken 頁複製 token：https://dashboard.ngrok.com/get-started/your-authtoken
3. 雙擊「**設定固定網址_ngrok.command**」，貼上 token，按 Enter

之後分享：
1. 雙擊「**固定網址分享水族箱.command**」
2. 視窗會顯示固定的 `https://...ngrok-free...` 網址，並自動複製到剪貼簿
3. 直接貼給朋友；控制台 QR Code 也會自動使用這個網域

注意：
- 視窗不能關，關了分享就斷；但下次重開仍會是同一個固定網址
- ngrok 免費版第一次打開網頁可能會出現提示頁，按 Visit 進入即可
- 免費方案有流量 / 請求量限制，適合活動前遠端測試，不建議長期公開
- 拿到網址的人也能開控制台刪魚，只給信任的人

### 臨時網址（備援，免帳號 cloudflared）

雙擊「**分享水族箱到網路.command**」→ 視窗會顯示一個 `https://xxx.trycloudflare.com` 網址（自動複製到剪貼簿），貼給朋友即可。
- 網址每次重開都會換新的，斷線就重新雙擊、發新網址
- 視窗不能關，關了分享就斷
- 拿到網址的人也能開控制台刪魚，只給信任的人

## 背景調校（讓魚永遠是主角）

控制台三個滑桿，邊看投影邊調：

- **背景暗度**：背景太亮太花就調低（0.6~0.85 通常最好），魚的顏色不動、只壓背景
- **背景對比**：背景灰濛濛就拉高一點
- **背景水流動感**：背景會像在水裡一樣輕輕晃動 + 水紋流動；投影看起來頭暈就調低，0 = 完全靜止
- 訣竅：小朋友的畫如果是**黑底/深色底**（如黑色海底拼貼），暗度可以開高一點沒關係；亮色背景就壓到 0.6 左右

## 前景圖（立體感）怎麼做

投影內建「前景水草裝飾」（控制台可關）。想要更高級的立體感，幫背景配一張**前景 PNG**：

1. 找一張海底照片當背景 → 上傳
2. 把照片裡「最靠近鏡頭的東西」（珊瑚、岩石、水草）摳出來存成去背 PNG：
   - **Photopea.com**（免費網頁版 Photoshop，推薦）：開原圖 → 套索/快速選取選下方物件 → Ctrl+J 複製新圖層 → 隱藏原圖層 → Export As PNG
   - **iPhone**：照片 App 長按主體 → 「拷貝/分享」就是去背 PNG（iOS 16+，物件多時不好控制）
   - **remove.bg**：網頁拖上去自動去背（免費有額度）
3. 控制台該背景縮圖按「🌿加前景」上傳 PNG
4. 魚游過會被前景擋住 → 立體
   - **前景 PNG 的畫布要和背景圖一樣大**（在 Photopea 同一檔案裡做就自動一樣），系統才能對位
   - 前景建議只留畫面下方 1/4～1/3 的物件，中間清空，不然會擋魚

## 投影質感建議

- 現場燈光調暗，投影對比拉高 → 光束和氣泡會非常明顯
- 投影機解析度設 1920×1080，瀏覽器務必全螢幕（F11）
- 背景圖選「深色系海底」最好看；亮色背景會吃掉光束效果
- 喇叭接筆電，環境音量小聲即可（背景襯托用）
